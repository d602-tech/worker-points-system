# 115年度綜合施工處職安環保協助員系統 — 維護更新日誌

## 📅 2026-04-30 更新記錄 (v3.2.x -> v3.2.5)

### 1. 差勤排班模組 (AttendanceSchedule)
*   **[修正] 調班持久化失敗**：
    *   **問題**：調班狀態在重新整理後會消失，無法正確與 Google Sheets 同步。
    *   **解法**：實作 `amStatus="調"` 的編碼邏輯，並透過備註欄位識別「對調日期」。在前端載入時增加 `parseStatus` 邏輯，自動從備註或修改原因中還原「調班」狀態。
*   **[新增] 排班天數上限驗證**：
    *   **問題**：協助員排定的上班天數可能高於行政院標準上班天數。
    *   **解法**：在儲存前增加驗證邏輯，若 `(總工時 / 8)` 超過 `標準天數`，則阻止送出並顯示提示。
*   **[優化] 請假時數自動計算**：
    *   **問題**：手動計算請假結束時間過於繁瑣。
    *   **解法**：改為自動計算。起始時間預設 `08:00`，根據選擇時數自動推算結束時間（自動跳過 `12:00-13:00` 午休時段）。
*   **[修正] 載入資料崩潰 (`rt.includes`)**：
    *   **問題**：當後端回傳非字串資料（如 null 或 0）時，前端調用 `.includes()` 會發生錯誤。
    *   **解法**：在所有字串方法調用前強制使用 `String()` 轉型，並強化 `try-catch` 以顯示具體錯誤原因（如權限不足）。

### 2. 日曆與總覽 (CalendarOverview)
*   **[新增] 當月未完成項目面板**：
    *   **問題**：使用者容易漏填每日填報或漏傳佐證檔案。
    *   **解法**：在日曆下方新增動態面板，列出所有「未填報出勤」、「已出勤但未傳佐證且點數為0」的過期日期，並提供快捷導航按鈕。
*   **[優化] 點數預估邏輯**：
    *   **修正**：點數預估現在會正確排除「合約生效前」的日期。

### 3. 月報填報 (MonthlyReport)
*   **[優化] 載入體驗**：
    *   **問題**：載入中頁面過於空白，使用者誤以為沒資料。
    *   **解法**：新增詳細載入說明文字，列出系統正在檢索的項（例行項目、點數彙整、附件等），並加入骨架屏 (Skeleton) 視覺效果。

### 4. 業務規則設定 (Contract Rules)
*   **[新增] 契約時間限制 (2026/04/22 起)**：
    *   **問題**：系統不應顯示或計算 2026/04/21 之前的點數或未完成項目。
    *   **解法**：在 `shared/domain.ts` 定義 `CONTRACT_START` 常數，並同步至：
        *   `AttendanceSchedule`: 限制標準上班日從 4/22 開始。
        *   `CalendarOverview`: 過濾 4/22 之前的未完成項目。
        *   `TodayTasks`: 禁止填報 4/22 之前的日期並顯示提示。
